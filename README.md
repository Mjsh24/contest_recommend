# 공모전 추천 시스템 프로젝트 🎯📅

## 📦 데이터셋 및 데이터 흐름
- [Contest Korea 공모전 데이터](https://www.contestkorea.com/)
  - 다양한 공모전 정보를 분야별로 분류하여 데이터베이스에 저장
  - 사용자는 관심 분야, 참가 대상, 상금 범위를 선택하여 맞춤형 공모전 추천을 받을 수 있음

## 🎯 목표
- 사용자가 입력한 필터 조건(분야, 참가 대상, 상금 범위)에 맞춘 공모전 추천 시스템 구현
- 공모전 추천 결과와 함께 사용자가 후기를 남기고, 다른 사용자의 후기를 확인할 수 있는 시스템 구축
- Flask 기반 백엔드 서버 설계 및 MySQL 연동

## ✅ 진행 과정
1. 데이터 수집 및 전처리:
   - Contest Korea에서 공모전 데이터를 수집하고, CSV 파일로 저장 후 DB에 삽입
   - 데이터는 분야별 테이블로 분리하여 관리 (예: 광고_마케팅, 과학_공학, 디자인 등)

2. 공모전 추천 시스템:
   - 사용자 입력 필터(분야, 참가 대상, 상금 범위)를 기반으로 맞춤형 공모전 추천 리스트 생성
   - 정규표현식 기반 상금 범위 필터링 로직 구현
   - Jinja2 템플릿을 통한 추천 결과 렌더링

3. 리뷰 기능 구현:
   - 추천 결과에 대해 사용자가 후기를 남길 수 있는 UI 구성
   - AJAX 요청을 통해 DB에 후기 저장 및 실시간 조회
   - MySQL DB에 리뷰 테이블 생성 및 데이터 관리

4. 템플릿 연동 및 데이터 전달:
   - Flask와 Jinja2를 통해 필터링 결과 및 리뷰 데이터를 HTML에 동적으로 전달
   - 사용자 인터페이스를 직관적으로 구성하여 사용자 경험을 최적화

## 📌 결론
- 필터 조건을 기반으로 맞춤형 공모전 추천을 제공하는 웹 시스템 구현에 성공함
- 사용자가 남긴 후기를 통해 다른 사용자에게 공모전 참여 경험을 공유할 수 있는 플랫폼 역할을 수행할 수 있음
- 데이터베이스 연동 및 Flask를 통한 백엔드 설계 경험을 쌓을 수 있었음

## 🚀 시사점
- 필터 조건이 늘어날 경우 현재의 if-else 구조는 한계가 있으므로, **동적 쿼리 생성 방식**으로 전환이 필요
- 후기 기능은 현재 텍스트 기반에만 국한되어 있으나, **별점, 이미지 리뷰, 공모전명 연동** 등으로 확장 가능성 존재
- 향후 공모전 추천을 AI 기반 모델로 전환하여, 사용자 활동 이력 기반의 **개인화 추천 기능 추가** 고려

## 🛠️ 개선점
- 상금 필터링 로직의 정규화 및 데이터 통합 구조 개선 필요
- 사용자 리뷰 저장 시 닉네임, 리뷰 내용 외에 **날짜 및 공모전명 저장** 추가 필요
- 프론트엔드와의 협업을 통해 **UI/UX 최적화 및 인터페이스 고도화** 계획
